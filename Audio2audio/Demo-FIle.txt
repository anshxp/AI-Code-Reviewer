     ```js
     app.get("/buses/verify", (req, res) => {
     ```

---

2. **Credentials / Cookies**

   * You are using:

     ```js
     credentials: "include"
     ```

     This means the browser will send cookies.
     If your backend doesn’t handle CORS correctly with credentials, you’ll get a 500.

     In Express, you must configure:

     ```js
     app.use(cors({
       origin: "http://localhost:5173", // or React dev server port
       credentials: true
     }));
     ```

---

3. **Backend Error on Verification**

   * If your backend checks for a JWT, session, or cookie but fails to parse it → it throws an error → 500.
   * Look at the backend logs when the request happens. That’ll tell you exactly why.

---

4. **Wrong URL or Port**

   * If your backend is running on a different port (e.g., `5000`), but your fetch is pointing to `3000`, that will fail.

---

### ✅ Quick Fix Steps

1. **Check Backend Route**

   * Make sure `GET /buses/verify` exists and doesn’t crash.
